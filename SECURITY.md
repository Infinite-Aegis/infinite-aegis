# Security Policy

## Reporting a Vulnerability

Please report suspected security vulnerabilities privately.

Do not open a public Issue, Pull Request, Discussion, or Discord message containing technical details of an unpatched vulnerability.

Reports may be sent by email:

**infiniteaegis.org@proton.me**

You may also contact the project administration through the Infinite Aegis Discord server:

https://discord.gg/t8XAuymcDs

When using Discord, contact a project administrator privately. Do not post vulnerability details in public channels.

## What to Include

Please include, when possible:

- a description of the vulnerability;
- affected files, systems, or versions;
- steps required to reproduce it;
- possible impact;
- logs, screenshots, or proof-of-concept information;
- any suggested mitigation.

Do not access, modify, or distribute data that does not belong to you.

Do not perform destructive testing against public project infrastructure.

## Disclosure

Please do not publicly disclose a vulnerability before the project team has had a reasonable opportunity to investigate and address it.

The project team may request additional information during the investigation.

## Scope

This policy covers security issues in Infinite Aegis code, project infrastructure, deployment configuration, and original project services.

Security issues in upstream Space Station 14, RobustToolbox, GitHub, third-party dependencies, or hosting providers may also need to be reported to the appropriate upstream maintainer or service provider.
